## 2.2

新功能：

- 支持打开 **Markdown（`.md`）** 文件：与 `.txt` 文件一样直接打开（不转换）；章节按 ATX `#` 标题识别，侧栏按标题层级缩进
- **编辑模式** 支持显示 **小地图**：「设置」→「编辑」可开启小地图；小地图支持显示大号章节名，便于在长文中定位章节

改进：

- **只读 ↔ 编辑视口恢复**：进入/退出编辑时以第二行视觉行为锚点进行滚动位置恢复，减轻切换后排版差异导致的跳动
- 编辑模式支持 **自动刷新章节**：内容变更时会同步更新章节列表，不需要手动「刷新章节」了（_出于性能考虑，内容大于 30 万行时，会禁用自动刷新，仍需手动刷新_）

修复：

- 关闭「行首缩进」后可正确恢复打开时的原始文本
- 短文本编码检测问题

## 2.1

改进：

- 流式加载 → 文本格式化 → 章节生成逻辑重构：
  - 加载期间不统计字数、不匹配章节，在加载完成后再统一处理，逻辑更简单清晰
  - 「压缩空行」「行首缩进」、章节匹配规则变更等，不再重新加载文件，直接基于内存中的物理文本进行操作，对大文件的体验有明显优化
- 切换「压缩空行」后恢复视口位置改为第二个视觉行
- 进入语音朗读时，从视口内第一行开始朗读（不再从中间行）
- 全文搜索：同一行多个匹配项现在会展示为多个搜索结果

## 2.0

新功能：

- **语音朗读**：支持 EdgeTTS、系统语音和 DashScope 朗读
- **编辑模式**：方便对小说内容进行修改（理论上也可以用来写小说）

- 设置弹框改版：改为多标签页形式，以支持以上新功能的设置

改进：

- Monaco 中文化
- 书签列表：显示章节名，编辑书签支持「更新为当前行」
- 信息栏：
  - 点击文件路径弹出菜单（在文件管理器中显示、重新加载、关闭文件）
  - 点击编码弹出菜单（保存为 UTF-8、保存为 GB2312）

## 1.4

新功能：

- 侧栏改版
  - 标签页改为类 VS Code 的展示方式（图标 + 面板）
  - 增加「搜索」标签页，支持在当前文件内搜索
  - 顶栏的「高亮词」弹框移至侧栏作为单独的标签页
  - 显示「配色」和「设置」快捷入口
- 文件列表右键菜单增强：分类、移除、重命名、在新窗口中打开
- 设置面板
  - 增加「同步当前文件」开关，默认关闭，开启后，如果当前正在阅读的文件被修改，将自动重新加载
  - 增加「平滑滚动」开关，默认开启，关闭后，阅读区滚动不再使用平滑动画，以减缓长时间阅读的视觉疲劳
  - 增加「章节最少字数」设置，默认 100，少于这个字数将不会被识别为章节

改进：

- 文件列表：显示特定分类时，「清空」按钮改为「清空分类」，交互上更符合直觉
- 文件列表编辑模式：编辑过程只写内存不落盘，退出编辑模式时再统一写入
- 高亮词列表：点击高亮词的跳转走内联搜索逻辑，不再走查找栏逻辑
- 底栏：文件大小改为打开时获取，以免文件更新后大小不准确
